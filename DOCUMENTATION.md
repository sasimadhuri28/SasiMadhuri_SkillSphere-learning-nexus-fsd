# Day 2: Git Commands

- **git init** – Creates a new Git repository in your project folder.
- **git clone** – Copies an existing repository from GitHub to your computer.
- **git add** – Adds a specific file to the staging area.
- **git add .** – Adds all changed files in the current folder to the staging area.
- **git commit -m "message"** – Saves a snapshot of the staged changes with a message.
- **git status** – Shows the current status of files.
- **git log** – Displays the history of all commits.
- **git diff** – Shows the differences between current changes and the last commit.
- **git config** – Sets Git configuration such as username and email.
- **git remote** – Shows or manages connections to remote repositories.
- **git push** – Uploads local commits to the remote repository.
- **git pull** – Downloads the latest changes from the remote repository and merges them.
- **git merge** – Combines changes from one branch into another.
- **git checkout** – Switches to another branch or restores files.
- **git switch** – Switches from one branch to another.
- **git branch** – Lists, creates, or deletes branches.
- **git reset** – Removes commits or unstages changes.
- **git revert** – Undoes a commit by creating a new commit.
- **git cherry-pick** – Copies a specific commit from one branch to another.
- **git rebase** – Reapplies commits on top of another branch.
- **git fork** – Creates your own copy of someone else's GitHub repository.

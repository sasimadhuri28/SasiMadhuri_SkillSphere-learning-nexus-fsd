GIT Commands

• git init – Creates a new Git repository in your project folder.
• git clone <repository-url> – Copies an existing repository from GitHub to your computer.
• git add <file-name> – Adds a specific file to the staging area (prepares it for commit).
• git add . – Adds all changed files in the current folder to the staging area.
• git commit -m "message" – Saves a snapshot of the staged changes with a message.
• git status – Shows the current status of files (changed, staged, or untracked).
• git log – Displays the history of all commits in the repository.
• git diff – Shows the differences between the current changes and the last commit.
• git config – Sets Git configuration such as username and email.
• git remote – Shows or manages connections to remote repositories like GitHub.
• git push – Uploads local commits to the remote repository.
• git pull – Downloads the latest changes from the remote repository and merges them.
• git merge – Combines changes from one branch into another branch.
• git checkout <branch-name> – Switches to another branch or restores files.
• git switch <branch-name> – Switches from one branch to another.
• git branch – Lists, creates, or deletes branches.
• git reset – Removes commits or unstages changes by moving Git back to an earlier state.
• git revert <commit-id> – Undoes a commit by creating a new commit that reverses its changes.
• git cherry-pick <commit-id> – Copies a specific commit from one branch to another.
• git rebase – Reapplies commits on top of another branch to maintain a clean history.
• git fork – Creates your own copy of someone else's GitHub repository (done on GitHub website).
